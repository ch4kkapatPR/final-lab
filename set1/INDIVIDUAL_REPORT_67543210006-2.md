## ขอบเขตงานที่รับผิดชอบ
ระบุส่วนงานที่ตนเองรับผิดชอบอย่างชัดเจน เช่น Auth Service, Task Service, Log Service, Docker Compose รวมไปถึงทดสอบระบบในฝั่งของตัวเอง
## สิ่งที่ได้ดำเนินการด้วยตนเอง
- การสร้าง Hash สำหรับรหัสผ่านของ seed data เพื่อให้สามารถใช้ login ได้
- การ Test E2E ร่วมกับทีม เพื่อแก้ไขปัญหาที่เกิดขึ้น
- การ Test API ในส่วนของการดึง การสร้าง การลบ การอัพเดต task ต่างๆ
- เพิ่ม logging helper

## ปัญหาที่พบและวิธีการแก้ไข
- การ login ไม่ได้ทั้งๆ ที่ hash แล้ว ซึ่งปัญหาอาจจะมาจากการที่ Generate Hash ออกมาแล้วรหัสผ่านไม่ตรงกับความเป็นจริง จึงได้ทำการ ReGenerate ใหม่ และเพิ่มการรีเฟรชตารางก่อนการรันสคริปต์ เพื่อไม่ให้เกิดปัญหาซ้ำซ้อน
- การ build หลังจากการแก้โค้ดและยังทำงานไม่ได้ทั้งที่โค้ดน่าจะไม่มีอะไรผิดพลาดแล้ว เป็นเพราะการที่ compose down เพียงอย่างเดียว volume อาจจะเหลือค้างทิ้งไว้จากการที่เก็บเป็น Cache โดยจะใช้ docker compose down -v เพื่อเคลียร์ออกให้หมด และทำการ build ใหม่ 

## สิ่งที่ได้เรียนรู้จากงานนี้
การใช้ Docker ในการทำงานเป็นหลัก เนื่องด้วยการทำงานคนละเครื่องกับทีม ทำให้สภาพเเวดล้อมในการพัฒนาอาจจะต่างกันสามารถทำงานไปพร้อมๆ กันได้ การแก้ไขปัญหาที่เกิดขึ้นในส่วนต่างๆ โดยการ log ออกมาว่าส่วนไหนทำงานผิดพลาดเพื่อให้สามารถแก้ไขได้อย่างตรงจุด และการทำงานร่วมกันเป็นทีม
